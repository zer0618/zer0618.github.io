坑爹网络配置涨姿势

Posted on 2018-10-07 | In linux

宿舍：

sudo dhcpcd eth0

首先得是动态获取IP地址 不然怎么都打不开认证的网 说明里还说没网络连接都能上呢

而且还说出现这种错误就去看…一个网站 我都上不去网怎么看啊摔 开玩笑的吧
s
udo openvpn –config login–bonn.conf #
然后用户名是z6+房间号 z6是我们宿舍的编号貌似

图书馆：

首先得修改无线网络配置

ctrl_interface=/var/run/wpa_supplicant

network={

    ssid="eduroam"
    key_mgmt=WPA-EAP
    proto=RSN
    eap=TTLS
#    anonymous_identity="anonymous@uni-***.de"
    identity="your-uni-id@uni-***.de"
    password="*******"
    ca_path="/etc/ssl/certs/"
    phase2="auth=PAP"

然后就可以：

sudo wpa_supplicant -B -i eth1 -Dwext -c /etc/my_wpa_supplicant.conf

无线网VPN：

下载了以后捏

tar -xzf  vpnclient-linux.tar.gz #解压
cd vpnclient
wget http://www.fseitz.de/download/vpnclient.patch-2.6.38  #patch
patch < vpnclient.patch-2.6.38
wget https://github.com/ruphert/CiscoVpnClient-Patches/raw/master/kernel-3.x-support.diff
patch -d vpnclient/ < kernel-3.x-support.diff 
sudo ./vpn_install
#启动
sudo /etc/init.d/vpnclient_init start

之后

Kopieren der Konfigurationsdatei in das richtige Verzeichnis:

cp unibn-vpn.pcf /etc/CiscoSystemsVPNClient/Profiles/

Die Zugriffsrechte sollten noch etwas angepaßt werden:

chmod 755 /etc/CiscoSystemsVPNClient

chmod 755 /etc/CiscoSystemsVPNClient/Profiles

chmod 644 /etc/CiscoSystemsVPNClient/Profiles/unibn-vpn.pcf

Auf- und Abbau der VPN-Verbindung (unter der normalen Benutzernummer):

vpnclient connect unibn-vpn

vpnclient disconnect
